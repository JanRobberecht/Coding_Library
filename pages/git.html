Open PowerShell and navigate to the folder where the code is located. Then execute all the commands below in order.

First time

TL;DR

git init
git config --local user.name "FirstnameLastname"
git config --local user.email "ma@mail.com"
git add --all
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/{user}/{reponame}.git
git push -u origin main

Explanation

git init

  • git → runs git
  • init → creates a new, empty Git repository in the current folder, so you can use that folder for version control.

git config --local user.name "FirstnameLastname"
git config --local user.email "ma@mail.com"

  • config → sets configuration options for Git. These options tell Git things like your identity, editor, merge tools, and other preferences.
  • --local → means these settings apply only to the repository you’re currently in. There are also --global (applies to your user account on the machine) and --system (applies to all users on the system). Setting it locally is useful if you want different identities for different repositories (for example, work vs personal).

  • add → stages changes (files) to be committed.
  • --all → stages all changes, including new, modified, and deleted files. Even outside the current folder while "git add ." only looks at the current folder and its subfolders.

git commit -m "first commit"

  • Creates a new commit with all files you have staged via "git add" and with the message you capture a snapshot of your project with a short description.
  • commit → records changes in your local repository.
  • -m → lets you write a message inline.
  • "first commit" → the commit message describing your changes.

git branch -M main

  • Renames the current branch to main, with -M forcing the change.
  • branch → used to create, list, rename, or delete branches in your repository.
  • -M → stands for “move/rename” and it forces the rename even if a branch with the target name already exists. It’s like a safe overwrite.
  • main → the new name you want for the local branch. In many modern Git setups, main is used instead of the old default master.

git remote add origin https://github.com/{user}/{reponame}.git

  • Adds a shortcut (named origin) that points to this GitHub repository URL, so Git knows where to push and pull changes from.
  • remote → manage connections to remote repositories.
  • add → create a new remote connection.
  • origin → the name for that remote (by convention, the main one is called origin).
  • URL → the address of the repository on GitHub.

git push -u origin main

  • push → uploads your local commits to a remote repository.
  • -u → it links your local branch (main) to your remote remote (origin) branch and sets it to default so future git push/git pull can be run without specifying the branch. It tells Git: "Remember that my local branch should track this remote branch."

Not first time

TL;DR

git pull
git status
git add --all
git status
git commit -m "A short menaingfull message to clarify what changes you made"
git push

Explanation

git pull

  • pull → fetches changes from a remote repository and merges them into your current branch.
  • git pull origin main → origin and main is not needed because you already set your current branch to an upstream (tracking) branch (git push -u origin main).
    You need to use it if (1) you haven't set up an upstream yet, (2) you want to pull a different branch than the one you are tracking or (3) you need to be explicit about which remote and branch to pull from.

git status

  • status → shows the current state of your repository: (1) which files are staged (ready to commit), (2) which files are modified but not staged, (3) which files are untracked (new files Git isn’t tracking yet) and (4) what branch you’re on and if it’s ahead/behind the remote.

git add --all

git status

git commit -m "A short menaingfull message to clarify what changes you made"

git push

  • git push -u origin main → see "git pull" why the "-u origin main" is not always necessary.
